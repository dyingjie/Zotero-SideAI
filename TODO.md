# Zotero-SideAI TODO

## 目标

开发一个适配 Zotero 8 的侧边栏 AI 插件，核心能力如下：

- 兼容 Zotero 8 插件机制
- 以侧边栏形式集成到 Zotero 界面中
- 支持自定义 `baseURL`
- 支持配置固定提示词（system prompt / 模板提示词）
- 支持自动读取并上传当前文本内容给模型
- 支持将模型返回结果展示在侧边栏中

## MVP 范围

第一版先只做最小可用闭环，不追求功能过多：

- 在 Zotero 侧边栏打开插件面板
- 用户填写 API Key、`baseURL`、模型名
- 用户保存固定提示词模板
- 插件自动获取“当前文本”
- 点击发送后，请求 AI 接口
- 在侧边栏展示回答结果和错误信息

“当前文本”第一版建议先明确为以下之一，并固定实现，不要一开始做太多来源：

- 优先方案：当前选中文献的标题 + 摘要 + notes
- 次选方案：当前选中的 note 全文
- 延后方案：PDF 全文抽取、选区文本、附件 OCR

## 技术设计建议

### 模块拆分

- [ ] `settings`：配置读取、保存、校验
- [ ] `context`：当前文本采集
- [ ] `prompt`：固定提示词与消息组装
- [ ] `api`：AI 请求封装
- [ ] `sidebar`：侧边栏 UI 与交互

### 先别做的功能

第一轮建议不要同时做这些，避免项目失控：

- [ ] 多轮会话
- [ ] 多供应商协议并行支持
- [ ] PDF 全文抽取
- [ ] 向量检索
- [ ] 本地模型适配
- [ ] 复杂模板市场
- [ ] 自动把结果写回 Zotero 条目

## 当前结论

这类插件最关键的不是“AI 能调用”，而是先把三件事做稳：

- Zotero 8 插件生命周期
- 侧边栏 UI 集成
- 当前文本采集和 Prompt 组装

只要这三块稳定，后面的 `baseURL`、模型切换、固定提示词都会比较顺。
